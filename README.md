# 100 Days of Python

A structured, beginner-friendly Python curriculum. One topic per day, real challenges every day, projects every few days. Start from zero — no experience needed.

## How to use this repo

1. **Fork it** — click "Fork" at the top right of this page so you have your own copy
2. **Clone your fork** — `git clone https://github.com/YOUR_USERNAME/python-100-days`
3. **Follow the days in order** — don't skip ahead; each day builds on what came before
4. **For each day:**
   - Read the `README.md` in the day folder first
   - Open `01_learn.ipynb` and work through it
   - Open `02_challenges.ipynb` and attempt every challenge
   - Check `notes.md` as a quick reference while you work
   - Only open `solution.py` after you've genuinely tried

5. **Project days** (marked with 🏗️) — read the brief, build it yourself, then compare to the solution folder

**Rule:** commit your work every day. Even if it's messy. The habit matters.

## What you need

- Python 3.8 or higher
- Jupyter Notebook or JupyterLab (for the `.ipynb` files)
- A code editor — VS Code is recommended

See [SETUP.md](SETUP.md) for installation instructions.

## Per-day structure

Every regular day contains:

```
Day_XXX_TopicName/
├── README.md            ← objectives and what to open first
├── 01_learn.ipynb       ← guided lesson with examples
├── 02_challenges.ipynb  ← 5 challenges rated ⭐ to ⭐⭐⭐
├── notes.md             ← quick-reference cheat sheet
└── solution.py          ← reference answers (check after attempting)
```

## Project days

Project days replace the learn/challenge notebooks with a build prompt:

```
Day_XXX_Project_Name/
├── README.md            ← project brief and requirements
├── starter.py           ← skeleton code to fill in
└── solution/
    └── project_name.py  ← one complete solution
```
