# Day 50 — Medium Interview Problems

## Complexity targets
| Problem | Time | Space |
|---------|------|-------|
| Longest substring | O(n) | O(k) |
| Container with most water | O(n) | O(1) |
| Three sum | O(n²) | O(1) |
| Coin change | O(n*amount) | O(amount) |
| Number of islands | O(rows*cols) | O(rows*cols) |
| Decode string | O(n) | O(n) |

## Problem-solving mindset
- State complexity before coding
- Draw examples on paper first for complex problems
- Start with brute force if stuck, then optimise
- Think about WHAT you need to store, not just HOW

## Interview tips
- Sliding window → Longest substring, minimum window
- Two pointers → Container with water, three sum
- Stack → Decode string, evaluate expression
- BFS/DFS → Number of islands, word ladder
